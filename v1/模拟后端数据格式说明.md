# 模拟后端数据格式说明

本文档详细说明了项目中模拟后端数据的格式和字段含义。

## 数据生成函数

项目使用 `src/utils/dataGenerator.js` 中的函数生成模拟数据：
- `generateSystemData()` - 生成完整的系统数据
- `generateMockBladeData(bladeId, status)` - 生成单个叶片数据
- `generateAttitude()` - 生成姿态数据

---

## 完整系统数据格式

调用 `generateSystemData()` 返回的数据结构如下：

```javascript
{
  system: {
    attitude: {...},           // 姿态数据
    environment: {...},         // 环境数据
    satelliteSignal: {...},     // 卫星信号数据
    relays: {...},              // 继电器数据
    controlUnit: {...},         // 控制单元数据
    alarmDetector: {...},       // 报警检测器数据
    displacementFault: {...},   // 位移故障数据
    highFrequencyFault: {...}   // 高频故障数据
  },
  blades: [...]                // 叶片数据数组
}
```

---

## 1. 系统数据 (system)

### 1.1 姿态数据 (attitude)

```javascript
{
  roll: Number,              // 横滚角（度），范围：-5 到 5
  pitch: Number,             // 俯仰角（度），范围：-5 到 5
  rollThreshold: 3.0,        // 横滚角阈值（度）
  pitchThreshold: 2.5,       // 俯仰角阈值（度）
  status: String,            // 状态：'normal' | 'fault'
  recoveryHint: Object | null // 恢复提示（仅在故障时存在）
}
```

**recoveryHint 对象结构**（仅在 `status === 'fault'` 时存在）：

```javascript
{
  rollDirection: String | null,    // 横滚调整方向：'left' | 'right' | null
  rollValue: Number,               // 横滚调整值（度）
  pitchDirection: String | null,   // 俯仰调整方向：'up' | 'down' | null
  pitchValue: Number               // 俯仰调整值（度）
}
```

**说明**：
- 当 `Math.abs(roll) > rollThreshold` 或 `Math.abs(pitch) > pitchThreshold` 时，`status` 为 `'fault'`
- `recoveryHint` 仅在故障时存在，提供调整方向和数值

---

### 1.2 环境数据 (environment)

```javascript
{
  rainSnowStatus: String,    // 雨雪状态：'clear' | 'rain' | 'snow'
  airTemperature: Number,     // 空气温度（℃），范围：15-35
  dishTemperature: Number     // 天线温度（℃），范围：20-35
}
```

**说明**：
- `rainSnowStatus` 随机生成，70% 概率为 `'clear'`，20% 概率为 `'rain'`，10% 概率为 `'snow'`

---

### 1.3 卫星信号数据 (satelliteSignal)

```javascript
{
  signalStrength: Number,     // 信号强度，范围：60-100
  signalQuality: Number       // 信号质量（%），范围：70-100
}
```

---

### 1.4 继电器数据 (relays)

```javascript
{
  heatingRelays: Array,       // 加热继电器数组，长度为 6
  alarmRelay: Object          // 报警继电器
}
```

**heatingRelays 数组**：
```javascript
[
  { status: 'on' | 'off' },   // 每个叶片对应的加热继电器状态
  ...
]
```

**alarmRelay 对象**：
```javascript
{
  status: 'on' | 'off'        // 报警继电器状态（有故障叶片时为 'on'）
}
```

---

### 1.5 控制单元数据 (controlUnit)

```javascript
{
  status: String,             // 状态：'normal' | 'fault'
  faultType: String | null    // 故障类型：'circuit_error' | 'communication_error' | null
}
```

**说明**：
- 50% 概率为故障状态
- 故障时，`faultType` 随机为 `'circuit_error'` 或 `'communication_error'`

---

### 1.6 报警检测器数据 (alarmDetector)

```javascript
{
  status: String,             // 状态：'normal' | 'alarming'
  alarmType: String | null    // 报警类型：'rain_snow' | 'fault' | 'other' | null
}
```

**说明**：
- 当存在故障叶片、雨雪天气或随机触发时，`status` 为 `'alarming'`
- `alarmType` 根据触发原因确定

---

### 1.7 位移故障数据 (displacementFault)

```javascript
{
  status: String              // 状态：'normal' | 'fault'
}
```

**说明**：
- 50% 概率为故障状态

---

### 1.8 高频故障数据 (highFrequencyFault)

```javascript
{
  status: String,             // 状态：'normal' | 'fault'
  faultDesc: String | null    // 故障描述（仅在故障时可能存在）
}
```

**可能的故障描述**：
- `'高频头电路异常，请检查电源与信号线连接'`
- `'高频头接收信号弱，请检查天线指向及遮挡物'`
- `'高频头温度过高，请检查散热情况'`
- `'高频头内部模块故障，请联系维护人员'`

**说明**：
- 50% 概率为故障状态
- 故障时，50% 概率包含 `faultDesc`

---

## 2. 叶片数据 (blades)

`blades` 是一个包含 6 个叶片对象的数组，每个对象结构如下：

```javascript
{
  bladeId: String,            // 叶片ID，格式：'BLADE_01' 到 'BLADE_06'
  status: String,             // 状态：'normal' | 'heating' | 'fault' | 'offline'
  temperature: {
    current: Number | null    // 当前温度（℃）
  },
  heater: {
    power: String,            // 加热器电源状态：'on' | 'off'
    powerLevel: Number        // 加热器功率等级：0-100（仅在 power 为 'on' 时有效）
  },
  faultInfo: Object | null    // 故障信息（仅在 status === 'fault' 时存在）
}
```

### 2.1 温度数据 (temperature)

**温度范围**：
- `normal` 状态：20-30℃
- `heating` 状态：25-35℃
- `fault` 状态：20-120℃
- `offline` 状态：`null`

### 2.2 加热器数据 (heater)

**功率状态**：
- `offline` 状态：`power = 'off'`, `powerLevel = 0`
- `heating` 状态：`power = 'on'`, `powerLevel = 50-100`
- 其他状态：`power = 'off'`, `powerLevel = 0`

### 2.3 故障信息 (faultInfo)

仅在 `status === 'fault'` 时存在：

```javascript
{
  faultType: String,          // 故障类型键名（如 'HEATER_SHORT_CIRCUIT'）
  faultCode: String,           // 故障代码（如 'HE001'）
  description: String,         // 故障描述
  severity: String,            // 严重程度：'low' | 'medium' | 'high'
  suggestion: String           // 处理建议
}
```

**故障类型代码对照表**：

| 故障类型键名 | 故障代码 | 说明 |
|------------|---------|------|
| `HEATER_SHORT_CIRCUIT` | `HE001` | 加热器短路故障 |
| `HEATER_OPEN_CIRCUIT` | `HE002` | 加热器断路故障 |
| `TEMP_SENSOR_ERROR` | `TE001` | 温度传感器错误 |
| `NO_RESPONSE` | `CM001` | 无响应 |
| `OVER_TEMPERATURE` | `TE002` | 超温 |
| `UNDER_TEMPERATURE` | `TE003` | 低温 |
| `COMMUNICATION_FAILURE` | `CM002` | 通信故障 |

---

## 3. 叶片状态分布

每次调用 `generateSystemData()` 时，6 个叶片的状态会从以下数组中随机分配：

```javascript
['normal', 'normal', 'normal', 'heating', 'fault', 'offline']
```

**说明**：
- 保证至少有一个 `normal`、`heating`、`fault`、`offline` 状态
- 状态会随机打乱后分配给 6 个叶片

---

## 4. 数据使用示例

```javascript
import { generateSystemData } from './utils/dataGenerator';

// 生成系统数据
const systemData = generateSystemData();

// 访问系统数据
console.log(systemData.system.attitude.roll);
console.log(systemData.system.environment.rainSnowStatus);
console.log(systemData.system.satelliteSignal.signalStrength);

// 访问叶片数据
systemData.blades.forEach(blade => {
  console.log(`${blade.bladeId}: ${blade.status}`);
  if (blade.faultInfo) {
    console.log(`故障: ${blade.faultInfo.description}`);
  }
});
```

---

## 5. 注意事项

1. **随机性**：所有数据都是随机生成的，每次调用 `generateSystemData()` 都会产生不同的数据
2. **数据关联**：某些数据之间存在逻辑关联：
   - 有故障叶片时，`alarmRelay.status` 为 `'on'`
   - 加热状态的叶片，对应的 `heatingRelays` 状态为 `'on'`
   - 报警检测器的状态会根据故障、雨雪等情况自动判断
3. **空值处理**：
   - `offline` 状态的叶片，`temperature.current` 为 `null`
   - 非故障状态的叶片，`faultInfo` 为 `null`
   - 正常状态的姿态数据，`recoveryHint` 为 `null`
4. **数值精度**：
   - 角度值保留 2 位小数
   - 温度值保留 1 位小数
   - 信号质量保留 1 位小数

## 6. 相关常量定义

数据中使用的状态和故障类型定义在 `src/constants/index.js` 中：

- `BLADE_STATUS` - 叶片状态常量
- `FAULT_TYPES` - 故障类型代码
- `FAULT_SUGGESTIONS` - 故障处理建议
